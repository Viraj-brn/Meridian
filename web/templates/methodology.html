Index Methodology

How the Meridian 50 Index is constructed, maintained, and analyzed

1. Index Construction

Universe Selection

The Meridian 50 Index comprises 50 large-cap US equities selected from the S&P 500, spanning all 11 GICS (Global Industry Classification Standard) sectors. Stocks are chosen for their market capitalization, liquidity, and sector representation to create a diversified benchmark.

Weighting Methodology

The index uses free-float adjusted market capitalization weighting, the same methodology used by MSCI World, S&P 500, and most major global benchmarks.

Weight_i = (Price_i × Shares_i × FFF_i) / Σ(Price_j × Shares_j × FFF_j) Where: Price_i = Current share price of stock i Shares_i = Total shares outstanding FFF_i = Free-float factor (0 to 1) Σ = Sum over all constituents j

The free-float factor excludes shares that are not available for public trading: government holdings, strategic cross-holdings, insider/founder stakes, and locked-up shares. This ensures the index reflects investable market capitalization only.

The Divisor Method

Index levels are computed using the divisor method, which ensures continuity across rebalancing events and corporate actions.

Index(t) = Σ(P_i(t) × Q_i × FFF_i) / Divisor Initial: Divisor = Σ(P_i(0) × Q_i × FFF_i) / Base_Level Rebalance: D_new = D_old × (New_Numerator / Old_Numerator)

The divisor absorbs all structural changes (additions, removals, corporate actions) so the index level reflects only market price movements, never artificial jumps from composition changes.

2. Rebalancing

The index rebalances quarterly (every 63 trading days, approximately matching MSCI's February, May, August, November schedule). During rebalancing:

  • Market capitalizations are updated using current prices
  • Weights are recalculated from the new market caps
  • The divisor is adjusted to maintain index continuity
  • Rebalancing history is logged for audit purposes

3. Risk Analytics

Volatility

We compute both rolling (simple moving window) and EWMA (exponentially weighted) volatility. EWMA gives more weight to recent observations, making it more responsive to changing market conditions.

Rolling Vol = std(r_t, ..., r_{t-w+1}) × √252 EWMA Vol = ewm_std(r_t, span=w) × √252 Where: r_t = log return on day t w = window length (20 or 60 days) 252 = trading days per year (annualization factor)

Value at Risk (VaR)

Historical VaR at the 95% confidence level: we sort historical daily returns and take the 5th percentile. If 1-day 95% VaR = 2%, then on 95% of trading days, the index loses less than 2%.

Conditional VaR (CVaR / Expected Shortfall) answers a deeper question: "Given that we're in the worst 5% of days, how bad is it on average?" CVaR is a coherent risk measure and captures tail risk that VaR ignores.

Maximum Drawdown

The largest peak-to-trough decline in the index level. This represents the worst cumulative loss an investor would have experienced during the observation period.

4. Performance Metrics

Sharpe Ratio
Risk-adjusted return: (annualized return - risk-free rate) / annualized volatility. A Sharpe ratio above 1.0 is considered good for a long-only equity index.
Sortino Ratio
Like Sharpe, but uses downside deviation instead of total volatility. Only penalizes negative returns — upside volatility is not a risk from the investor's perspective.
Tracking Error
The annualized standard deviation of the difference between the index return and the benchmark (SPY) return. Low tracking error means the index closely follows the benchmark.
Information Ratio
Active return divided by tracking error: IR = (R_p - R_b) / TE. Measures how much excess return you get per unit of active risk.

5. Factor Analysis

We perform Principal Component Analysis (PCA) on the return covariance matrix to extract the dominant risk factors driving index returns. This is a simplified version of MSCI's Barra factor model.

Covariance Matrix: Σ = P · Λ · P^T (Eigendecomposition) Λ = diagonal matrix of eigenvalues (variance per factor) P = matrix of eigenvectors (the principal components) Factor 1 ≈ "Market" (all stocks move together) Factor 2 ≈ "Sector rotation" or "Growth vs Value" Factor 3+ ≈ More granular risk dimensions

The first principal component typically explains 40-60% of total portfolio variance, confirming that "the market" is the dominant risk factor. Subsequent factors capture sector tilts, style exposures, and other systematic risk dimensions.

6. Technology Stack

  • Python — Core computation (NumPy, Pandas, SciPy)
  • SQLite — Persistent storage with SQL analytics
  • matplotlib — Professional chart generation
  • Flask — Lightweight web server
  • yfinance — Market data source
  • scikit-learn — PCA for factor decomposition

7. Data Quality

Data integrity is paramount — at MSCI's scale, a single pricing error can misrepresent billions in benchmarked assets. Our data pipeline includes:

  • Validation: Reject negative prices, flag zero prices
  • Forward-fill: Handle missing trading days
  • Anomaly detection: Flag >20% daily moves as potential corporate actions
  • Gap detection: SQL queries to find missing date ranges
  • Logging: All anomalies logged with timestamps
